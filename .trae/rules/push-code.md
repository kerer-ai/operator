# 代码推送规则

## 描述
此规则确保每次代码修改都会触发 README 刷新技能，执行 AI 检查，并处理自动提交到远程仓库的操作。

## 工作流程

1. **代码修改检测**
   - 监控代码文件的变化
   - 当检测到代码修改时触发此规则

2. **README 刷新**
   - 自动调用 `readme-refresher` 技能
   - 路径：`.trae/skills/readme-refresher/SKILL.md`
   - 该技能将分析代码变化并相应更新 README 文件

3. **AI 检查**
   - 对更改执行基于 AI 的审查
   - 生成检查意见和建议
   - 关注文档准确性和代码质量对齐

4. **用户交互**
   - 向用户显示检查结果
   - 提示用户确认是否应用建议的修改
   - 允许用户在继续之前查看和调整更改

5. **自动提交**
   - 用户确认后，暂存所有更改
   - 创建有意义的提交消息，总结更改内容
   - 将提交推送到远程仓库

## 触发条件
- 当代码文件被修改时
- 在 git 提交操作之前
- 在推送到远程仓库的操作时

## 优势
- 确保文档与代码保持同步
- 提供 AI 驱动的质量检查
- 简化提交过程
- 保持项目文档的一致性

## 实施说明
- 此规则应作为开发工作流程的一部分自动执行
- 确保 git 操作的适当权限
- 安全配置远程仓库凭证
- 处理过程中可能出现的任何合并冲突
